# degens

March Madness mobile-first reference board built with Vite + React.

## What This App Does

- shows the current tournament board
- supports `All`, `Today`, `Tomorrow`, `Live`, and `Final` filters
- displays active model predictions, market lines, and projected/actual bet categories
- surfaces model accuracy for winner, cover, failed to cover, over, and under

## Local Development

```bash
npm install
npm run dev
```

Build:

```bash
npm run build
```

## Data Source

The app reads from:

```text
public/data/current-board.json
```

That file is generated by:

```bash
node ./scripts/export-current-board.mjs
```

The export script reads from the MySQL tournament backend and writes the current frontend snapshot.

## Required Env Vars For Data Export

If you want to regenerate `public/data/current-board.json`, the exporter needs DB access:

```env
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3307
MYSQL_DB=jonnyrosero_march_madness
MYSQL_USER=jonnyrosero_mlb
MYSQL_PWD=...
```

If omitted, the script defaults to the local SSH-tunneled MySQL setup above.

## Vercel Deploy

Recommended Vercel settings:

- Framework Preset: `Vite`
- Build Command: `npm run build`
- Output Directory: `dist`

Current deployment model:

- Vercel hosts the frontend
- backend refresh jobs stay outside Vercel
- the frontend snapshot must be refreshed separately when board data changes

## Backend Refresh Cadence

Operational cadence for the backend:

- once per day: schedule + odds + predictions
- recurring during games: live score refreshes

Current scripts that drive that flow live outside this repo.
